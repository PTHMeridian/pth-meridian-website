02
Why post-quantum — now

The threats are real

These are not theoretical future risks. They are active threats that post-quantum cryptography defeats.

Harvest Now Decrypt Later
HNDL Attack

Adversaries are recording encrypted traffic today — government communications, financial data, health records — to decrypt when quantum computers arrive. Data encrypted today with classical algorithms is already compromised.

✓ Defeated by ML-KEM
Signature Forgery
Quantum Forgery

Quantum computers running Shor's algorithm can forge digital signatures based on RSA and ECDSA — the algorithms protecting software updates, financial transactions and legal documents worldwide.

✓ Defeated by ML-DSA
Key Compromise
Vault Brute Force

GPU-accelerated attacks against cryptographic key vaults using iterative hash functions. Modern GPU clusters can test billions of password combinations per second against weakly derived keys.

✓ Defeated by Argon2id
Identity Fraud
Proof Substitution

Zero-knowledge proof systems with unbound range proofs allow attackers to substitute valid proofs for different claims — proving age eligibility with a proof generated for a different age range.

✓ Defeated by ZKP v2
Insider Threat
Single-Party Control

Single administrators with unilateral key access represent catastrophic insider threat vectors. One compromised or malicious person can sign fraudulent certificates, issue unauthorized credentials or destroy audit records.

✓ Defeated by Threshold Signatures
Certificate Fraud
Revocation Delay

Certificate Revocation Lists downloaded periodically leave windows of hours where revoked certificates appear valid. An attacker whose credentials are revoked at 9am can still authenticate at 9:02am against a cached CRL.

✓ Defeated by OCSP — 0ms delay

The timeline
The clock is already running.

NIST published the post-quantum cryptographic standards in 2024. The US government has mandated migration for all federal agencies by 2030. Canada's Communications Security Establishment is actively working on transition requirements.

Most organizations are not ready. The average enterprise migration takes three to five years. Organizations that start in 2027 will not finish by 2030. The window is now.

PTH Meridian has already implemented the standards. The code is open source. The audit trail is public. The work is done.
